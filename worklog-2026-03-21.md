## 项目说明（README）

基于 [LuxTTS](https://github.com/ysharma3501/LuxTTS) 的 Docker 容器化 API 服务，支持 Docker 和 Podman。

### 特性

- **GPU 加速**：PyTorch CUDA (cu128)，支持 RTX 4090/5080 等
- **离线运行**：模型构建时打包进镜像，运行时完全离线
- **输出音频**：48kHz WAV
- **显存占用**：约 1GB VRAM

### 前置要求

- NVIDIA GPU
- 已安装 [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html)
- Docker（含 Docker Compose）或 Podman

### 快速开始

#### Docker

```bash
git clone https://github.com/nijisakai/LuxTTS.git
cd LuxTTS
docker compose build && docker compose up -d
docker logs -f luxtts-gpu
```

#### Podman

```bash
git clone https://github.com/nijisakai/LuxTTS.git
cd LuxTTS
bash podman-run.sh
```

启动后访问：`http://localhost:9880/?text=你好&speaker=audio/花魁.wav`

### API 接口

**服务地址**：`http://localhost:9880`

```
GET  http://localhost:9880/?text=要合成的文本&speaker=audio/参考音频.wav
POST http://localhost:9880/  (支持 form 和 JSON body)
```

#### 参数说明

| 参数 | 必填 | 类型 | 默认值 | 说明 |
|------|------|------|--------|------|
| `text` | 是 | string | — | 待合成文本 |
| `speaker` | 是 | string | — | 参考音频路径（相对于项目根目录） |
| `num_steps` | 否 | int | `4` | 采样步数，值越大质量越好但更慢（推荐 3-4） |
| `guidance_scale` | 否 | float | `3.0` | 引导尺度，控制与参考音色的相似度 |
| `t_shift` | 否 | float | `0.5` | 采样温度，越高质量越好但可能有发音错误 |
| `speed` | 否 | float | `1.0` | 语速控制（1.0=正常，>1.0=加速，<1.0=减速） |
| `rms` | 否 | float | `0.01` | 音量控制，越大越响（推荐 0.01） |
| `ref_duration` | 否 | int | `5` | 参考音频最大时长（秒），降低可加速推理，如有瑕疵可设为 1000 |
| `return_smooth` | 否 | bool | `false` | 平滑模式，听到金属音时可设为 true |

> 所有可选参数不传时使用环境变量默认值。

**返回**：WAV 音频流（48kHz）

#### 使用示例

**浏览器直接访问：**
```
http://localhost:9880/?text=你好,测试一下&speaker=audio/花魁.wav
```

**GET 全参数示例：**
```
http://localhost:9880/?text=你好世界&speaker=audio/花魁.wav&num_steps=4&guidance_scale=3.0&t_shift=0.9&speed=1.0&rms=0.01&ref_duration=5&return_smooth=false
```

**curl（GET）：**
```bash
curl "http://localhost:9880/?text=你好世界&speaker=audio/花魁.wav" -o output.wav
```

**curl（POST JSON 全参数）：**
```bash
curl -X POST http://localhost:9880/ \
  -H "Content-Type: application/json" \
  -d '{
    "text": "你好世界",
    "speaker": "audio/花魁.wav",
    "num_steps": 4,
    "guidance_scale": 3.0,
    "t_shift": 0.9,
    "speed": 1.0,
    "rms": 0.01,
    "ref_duration": 5,
    "return_smooth": false
  }' \
  -o output.wav
```

**PowerShell：**
```powershell
Invoke-WebRequest "http://localhost:9880/?text=你好世界&speaker=audio/花魁.wav&speed=1.2&t_shift=0.9" -OutFile output.wav
```

**健康检查：**
```bash

# 返回: {"status": "ok", "device": "cuda"}
```

## 问题与修复记录

### 1. PyTorch 不识别 `npu` 设备 (RuntimeError)

**现象**：启动后请求 API 报错
```
RuntimeError: Expected one of cpu, cuda, ipu, xpu, mkldnn, ... device type at start of device string: npu
```

**原因**：`device="npu"` 时，`process_audio()` 中调用 `.to("npu")` 失败。PyTorch 没有原生 `npu` 设备支持，NPU 推理由 ONNX Runtime + OpenVINO 处理，PyTorch 张量应在 CPU 上。

**修复**：`zipvoice/luxvoice.py` 新增 `self.torch_device` 属性，`npu` 映射为 `cpu`，`encode_prompt()` 传给 `process_audio` 的 device 改用 `torch_device`。

**提交**：`eacb14b` — fix: NPU模式下PyTorch device映射为cpu，避免RuntimeError

---

### 2. 安装脚本缺少 `openvino` 依赖

**现象**：`onnxruntime_providers_openvino.dll` 加载失败，提示依赖 `openvino.dll` 缺失。

**修复**：`install-npu-windows.bat` 新增步骤 `[5/6] pip install openvino`。

**提交**：`c238ac0` — fix: NPU安装脚本增加openvino依赖

---

### 3. `pip install --upgrade pip` 报错

**现象**：
```
ERROR: To modify pip, please run the following command:
C:\Users\niji\LuxTTS\.venv\Scripts\python.exe -m pip install --upgrade pip
```

**修复**：`install-npu-windows.bat` 中 `pip install --upgrade pip` → `python -m pip install --upgrade pip`。

**提交**：`b0882d7` — fix: NPU安装脚本使用python -m pip升级pip

---

### 4. `start-npu.bat` 中 `&` 被当作命令分隔符

**现象**：
```
启动后访问: http://localhost:9880/?text=你好
'speaker' is not recognized as an internal or external command
```

**原因**：batch 中 `&` 是命令分隔符，`echo ...&speaker=...` 被拆成两条命令。

**修复**：简化 echo 中的 URL，去掉 `&speaker=...` 部分。

**提交**：`bad694e` — fix: start-npu.bat转义&符号避免被当作命令分隔符

---

### 5. OpenVINO DLL 找不到 + 端口被占用

**现象**：
```
Error loading "onnxruntime_providers_openvino.dll" which depends on "openvino.dll" which is missing
以一种访问权限不允许的方式做了一个访问套接字的尝试。
```

**修复**：`start-npu.bat` 中：
- 启动时自动将 `openvino/libs` 目录加入 PATH
- 启动前自动杀掉占用 9880 端口的旧进程

**提交**：`3a5ffb9` — fix: start-npu.bat添加OpenVINO PATH和端口冲突处理

---

### 6. `hf-mirror.com` SSL 连接失败

**现象**：
```
SSLError: [SSL: UNEXPECTED_EOF_WHILE_READING] EOF occurred in violation of protocol
HTTPSConnectionPool(host='hf-mirror.com', port=443): Max retries exceeded
```

**原因**：Windows 原生环境可直连 huggingface.co，不需要镜像。镜像站可能被 Clash 代理干扰。

**修复**：`start-npu.bat` 移除 `HF_ENDPOINT=https://hf-mirror.com`，注释保留供需要时启用。

**提交**：`fbe3a39` — fix: start-npu.bat移除HF镜像(Windows可直连HuggingFace)

---

### 7. `openvino` 与 `onnxruntime-openvino` 版本不匹配

**现象**：
```
Error loading "onnxruntime_providers_openvino.dll" But no dependency issue could be determined
```

**原因**：`openvino==2026.0.0` 与 `onnxruntime-openvino==1.24.1` 不兼容。ORT 1.24.1 需要 OpenVINO 2025.x。

**修复**：`install-npu-windows.bat` 中限定版本 `openvino>=2025.1,<2026`。

**提交**：`7345466` — fix: 固定openvino版本2025.x匹配onnxruntime-openvino 1.24

---

### 8. NPU 设备不可用（待装驱动）

**现象**：
```
[OpenVINO] Device NPU is not available
```

**原因**：Windows 上未安装 Intel NPU 驱动。

**状态**：待安装。驱动下载地址：https://www.intel.com/content/www/us/en/download/794734/intel-npu-driver-windows.html

**当前行为**：OpenVINO 创建失败后自动回退到 CPU ONNX 推理，服务功能正常，只是没有 NPU 加速。

## 安装后网络报错备注

安装过程中出现 `tls handshake eof` 错误（PyTorch whl/cpu 源 和 k2-fsa.github.io），是 Clash 代理干扰 HTTPS 导致。关闭全局代理后重装可解决。已安装的包不受影响。

## 已知问题

- 使用 `shoushou.wav` 作为参考音频时，合成语音开头部分音频会丢失。其他参考音频（花魁、厨娘、大叔等）未出现此问题。**解决方案**：将 `ref_duration` 设为 `1000`（即不截断参考音频）后恢复正常。原因是默认 `ref_duration=5` 截断后 prompt 编码质量下降，导致过渡区吞掉开头文字。
